# Creating a defect from Support Cases

While working on a **Support Case** you will **need to create defects** from time to time.

Either you found a **workaround **and just want to **register the defect** or you actually do a **fix **and **need to commit** it.
# **We need to keep all these systems linked**
Why?

- 
The **priority of a defect is influenced** by the number of associated **Support Cases**

- 
By following the **linked issues** we can find **more information**

- 
Sauron **centralizes all defect prioritization** on R&amp;D

# The links are automatic if you create the issue on Sauron
In this screen a template is provided for the description:

**Symptoms**&nbsp;

Description of what a one can observe as a result of this defect

**Steps to reproduce**

Step by step instruction on how to reproduce the defect. Attach files if you have any

**Cause**

Technically what causes the bug.

Architecture problem, bug in the source code etc...

**Workaround**

What steps can be followed to overcome the consequences of the defect without fixing it.

**Fix information**

What was changed to fix this problem

If you provide the **Support Case Jira Key** in the Sauron &ldquo;**Add Issue**&rdquo; screen both the Sauron and Jira defects are created and linked

Also, the Jira Support Case and Defect issues are linked
