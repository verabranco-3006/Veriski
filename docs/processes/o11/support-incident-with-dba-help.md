# Support Incident with DBA help

# Context
In an attempt to measure the impact DBAs are having in solving Support Incidents we want to understand what are and how many times a DBA help is needed.

To be able** to achieve this we will need the help of everyone** that is dealing with support cases and requests help from our DBAs to follow the process described below.

For the consolidated information you can see the dashboard here (soon).

You can see the list of support incidents that required the help of a DBA in the [Jira Filter](https://outsystemsrd.atlassian.net/issues/?filter=33902&amp;atlOrigin=eyJpIjoiNDQ3OTU0MzE4ZDk4NDRkODg3ZGVkOTQxMDA5NjEyNDYiLCJwIjoiaiJ9).
# Process
When the help of a DBA is needed in context of a support incident, you should:
- 
Contact the appropriate DBA
- 
See: How can you engage with the DBAs [https://outsystemsrd.atlassian.net/wiki/spaces/RT/pages/1667630030/R+D+DBA+Team#How-can-you-engage-with-the-DBAs%3F](https://outsystemsrd.atlassian.net/wiki/spaces/RT/pages/1667630030/R+D+DBA+Team#How-can-you-engage-with-the-DBAs%3F)

- 
Mark the support case with the label **dba_help**
- 
In the Support Incident jira page, press &ldquo;**.**&rdquo;

- 
Search &ldquo;**Labels**&rdquo; in the dialog

- 
Select the option Labels in the EDIT FIELDS

- 
Add the label **dba_help**

- 
Add the comment to the support incident &ldquo;**Requested DBA help (&lt;DBA Name&gt;).**&rdquo;
- 
You **MUST** write this comment everytime you request DBA help.

- 
If the Support Incident is waiting for the DBA input it should be placed in the **Waiting For Internal** Status.

# FAQ
When should I mark a Support Incident needed DBA help?

- 
Every time you have any question, ask for clarification or advice from a DBA you should mark the support incident with the label dba_help.
