# Support Cases High Level Lifecycle

This page describes at a **high level** the journey a Support Case goes through **since it is opened until a solution is provided**.

Please note that this is a **simplification** intended to show the flow of a Support Case that requires **R&amp;D intervention and/or a fix** to be solved.

A Support Case can be **solved at any of the Technical Support Lines**, only a **small subset of Support Cases reach R&amp;D.**

In the image above, we can see that:

- 
Support Cases are **initially **handled by the **1st Line** of Support, where they collect information from the reported issue and register it. If the customer is reporting a known issue to which there is a known solution in the **Support Knowledge Base**, the 1st Line will Close the Case.

- 
If the **1st Line is not able to solve** the Support Case, it will be **escalated to the 2nd and 3rd lines**. The 3rd Line is staffed by the most **experienced Support Engineers**.

- 
**Support investigates** the reported issue and will provide **workarounds** for the reported issue if they can do so.

- 
Support becomes **Blocked** when they are not able to provide any **solution** to the customer.

When **Support is Blocked**, **R&amp;D is required to intervene** in the Support Case.

In the second image we can see that:

- 
The 3rd Line of Support **escalates** the Support Case to R&amp;D.

- 
R&amp;D starts by doing the same thing Support does, **investigate** the reported issue, and trying to find a **workaround.**

- 
If **no workaround** can be found, **R&amp;D **[**produces a fix and delivers it**](https://outsystemsrd.atlassian.net/wiki/spaces/RRT/pages/364183707/How+to+release+a+QuickFix).

- 
**R&amp;D is never responsible for interacting with the customer**, although we can talk with the customer by requesting the Support Engineer to arrange a remote call. (It is useful in the most complex cases) 

Continue reading about how Support Cases reach the right R&amp;D team [here](https://outsystemsrd.atlassian.net/wiki/spaces/RCP/pages/726171771/Support+Case+distribution+Inside+R+D).
