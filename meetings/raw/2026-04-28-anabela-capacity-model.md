## Key Takeaways

### 1. Artifacts Used by Dev Teams

Anabela confirmed the artifacts dev teams work with for capacity and planning. Need to understand how these map to the capacity model's data layer.

### 2. Estimation Framework

Discussion on how dev teams estimate work — understanding their unit of measurement and estimation cadence to determine compatibility with the SP-based phase model.

### 3. Connection with Roadmap

Strong interest in connecting capacity to roadmap — specifically answering "when will this user story be ready?" This maps directly to the schedule and roadmap views already built for PE.

### 4. Gap Identified — Cross-Team/Asset Dependencies

**Key gap:** the current model operates within a single team boundary. Dev teams need visibility on **dependencies between teams and assets** — when Squad A is blocked by Squad B, or when a shared asset creates a bottleneck across multiple teams.

This is not covered by the current PE model (PE has no cross-team dependencies at this scale). Scaling to dev teams requires a dependency layer on top of the capacity model.
