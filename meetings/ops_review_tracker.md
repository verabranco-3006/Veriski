# Operations Review Tracker

*Bi-weekly meeting where practice leaders share operational results*

---

## Meeting Context

**Frequency:** Every 2 weeks
**Attendees:** SRE (Incident Response), Quality Team (Quality Practices), Process Engineering (Change Management, Retrospectives, Failure Management)

**Purpose:**
- Share 15-day reviews on each process
- Present results and action plans
- Create awareness of high-stake moments (incidents, initiative launches)
- Track operational metrics and trends

## Report Structure Template

Use this structure when preparing Process Engineering content:

### **Retrospective Management Data**
- Link to Power BI report
- Key observations (volume, trends, pending reviews)

### **Change Management Data**
- Link to Power BI report
- Executive summary (1-2 paragraphs with key findings)

#### Deep Dive Sections:
1. **Volume & Ring Distribution**
   - Data observations
   - Topic insights

2. **Risk Profiling**
   - Data observations
   - Topic insights

3. **Operational Usage & Category Distribution**
   - Data observations
   - Topic insights

#### **Highlights & Insights**
- Bulleted list of top findings with data and implications
- Action items with status (to be started, ongoing, completed)

### **Process Callouts**
- Major initiatives or transformations underway
- Policy updates or changes

### **Action Items for Process Engineering**
- Clear next steps with owners

- [ ] Update Confluence page with any new insights
